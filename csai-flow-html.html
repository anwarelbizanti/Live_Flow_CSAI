  • Receives ad request from player

  • Fires tracking events (start, quartiles, complete)

Key Differences: CSAI vs SSAI

CSAI Characteristics:

  • ‚úì Same content stream for all viewers
  • ‚úì Better CDN cache efficiency
  • ‚úì Lower infrastructure costs
  • ‚úì Personalized ads per viewer
  • ‚úó Ads can be blocked (ad blockers)
  • ‚úó Visible buffering during transitions
  • ‚úó Requires robust client SDK
  • ‚úó More complex player implementation

SSAI Characteristics:

  • ‚úì Seamless ad insertion (no buffering)
  • ‚úì Ad-blocker proof
  • ‚úì Simpler player implementation
  • ‚úì Better user experience
  • ‚úó Unique stream per viewer (CDN impact)
  • ‚úó Higher infrastructure costs
  • ‚úó More complex server architecture
  • ‚úó Potential latency added

Player Flow:

  1. Player parses manifest, detects cue point at timestamp T
  2. At time T-2s (pre-roll buffer), player makes ad request
  3. Ad SDK requests VAST from ad server with device/user info
  4. Ad server responds with ad creative URLs
  5. At time T, player pauses content stream
  6. Player loads and plays ad(s) from separate URL
  7. Player fires tracking beacons (impression, quartiles)
  8. After ad(s) complete, player resumes content stream

When to Use CSAI vs SSAI

Use CSAI When:

  • Budget-conscious operation
  • Moderate scale (thousands of viewers)
  • Can tolerate slight buffering
  • Need quick time-to-market
  • Platform SDKs are robust (Roku, Apple TV)
  • Ad blocking isn't a major concern

Use SSAI When:

  • Premium viewing experience required
  • Large scale (millions of viewers)
  • Must prevent ad blocking
  • Live sports or news (seamless experience)
  • Have budget for infrastructure
  • Need DVR with ads intact

üí° Hybrid Approach (Best of Both)

Some platforms use a hybrid model:

  • SSAI for premium content: Live sports, breaking news, premium shows
  • CSAI for standard content: VOD catalog, user-generated content
  • Device-dependent: SSAI for web browsers (ad blocking risk), CSAI for CTV apps (trusted environment)
  • Start with CSAI for MVP, migrate to SSAI as you scale and revenue justifies it
